# AGENTS.md — aiswitch

## Commands

```bash
npm run build          # tsc → dist/
npm test               # jest (test/**/*.test.ts)
npm run lint           # eslint src/**/*.ts test/**/*.ts
npm run format         # prettier --write
npm run format:check   # prettier --check
npm run verify         # build → lint → format:check → test → audit
```

Pre-commit runs `npm run verify`. Husky is configured via `prepare` script.

## Architecture

- Entry: `src/index.ts` → `src/cli.ts:runCli()`
- Commands:
  - TUI: `src/commands/select.ts` (main menu: Resume/Start/Create)
  - CLI: `src/commands/{run,list,which,doctor,init,create,resume,start,new}.ts`
  - Wrappers: `src/commands/{resume,start,new}.ts`
- Config: `src/config/{schema,paths,load}.ts` — loads `~/.aiswitch/config.yaml`
- Runtime: `src/runtime/{spawn,resolveExecutable,env,signals}.ts`
- Utils: `src/utils/{harness,detect-harnesses,extract-session,yaml}.ts`
- Tests: `test/*.test.ts`

Config schema (`src/config/schema.ts`):

- `version: 1`
- `profiles: { <name>: { executable, cwd?, args?, env?, description?, createDirs? } }`

Session schema (`src/commands/sessions.ts`):

- `id`: Harness session ID
- `profile`: Profile name
- `sessionKey`: User-friendly key (default: `<profile>_<last-4>`)
- `description?`: Optional description
- `cwd?`: Working directory
- `lastUsed`: Timestamp

Env vars with path-like values (`HOME`, `XDG_*`, `*_HOME`, `*_CONFIG_DIR`) get `~` expanded.

## Conventions

- TypeScript: ES2022, CommonJS, strict mode, `dist/` output
- Lint: `@typescript-eslint`, `no-console: off`, `no-explicit-any: warn`
- Format: Prettier, single quotes, 100 char width, 2-space tabs
- Test: Jest with `ts-jest`, node environment

## Gotchas

- `package.json:bin` points to `dist/aiswitch.cjs` (not auto-generated by tsc)
- `tsconfig.json` excludes `test/` — tests use ts-jest directly
- CLI passes args after profile name to child process; `--` separator optional
- **Tests MUST use `test/utils.ts`** — see `test/TEST-UTILS.md` for isolation patterns
- **NEVER touch `~/.aiswitch` in tests** — always use `useTestEnv()` helper
- TUI does NOT prompt for extra args (keep flow simple)
- Session save is manual (prompt after harness exits)

## Code Quality Rules

- **DRY**: Extract repeated logic to `src/utils/` (e.g., `yaml.ts` for profile serialization)
- **Error messages**: Include actionable suggestions (available profiles, commands to run)
- **Tests**: Cover commands, runtime utils, and edge cases; use temp directories for file I/O
- **Flags**: Support both short (`-f`) and long (`--force`) forms; add `--json` for machine-readable output
- **Config**: Respect `AIUSE_CONFIG` env var override in all commands
- **Env vars with path-like values**: `HOME`, `XDG_*`, `*_HOME`, `*_CONFIG_DIR` get `~` expanded
- **No hardcoded harnesses**: Never hardcode harness names (opencode/codex) in prompts/text. Use `src/utils/harness.ts` and `src/utils/detect-harnesses.ts` for harness-aware messages. Harness detection only for PATH checks.
